# Agent toolkit

Portable instructions for this user, organised as **skills** under [`skills/`](skills/) — the rules live in the skills, not in this file. The device's `~/.claude/CLAUDE.md` imports this; the toolkit is a git repo that travels across machines, so nothing here depends on device config.

A session learning worth keeping is codified into a skill — grow an existing skill, or add a new one only for a genuinely new core area (don't split skills too fine). Never duplicate a rule into device-local config (`~/.claude/*`) or per-repo docs; one home per rule. Propose the addition to the user — their toolkit, their call — write it once approved, and remind them to commit/push the toolkit so the learning isn't lost.
